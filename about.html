Sendmail Sobriquet was born out of the frustration of managing many hundreds of email aliases for my personal Sendmail server installation. As it happens, I value my privacy and like to create a new email for every company that asks me for my email address. If When I start to receive 3rd-party spam at one of those made-up email addresses, I immediately know who sold/disclosed/shared/leaked my email and I can respond appropriately. I create the new email addresses as aliases to a real email address by adding entries to Sendmail's alias database, which was (be default) set up as a simple text file: /etc/mail/alaises.

Many years went by, and several hundreds of aliases were added to /etc/mail/alaises. I grew increasingly tired of this process:

  1. ssh to mail server
  2. edit /etc/mail/aliases (add, remove, etc.)
  3. newaliases (to compile the text file into a binary database)
  4. exit

I really wanted an easier way for me, and even my family, to add and remove aliases.

Finally I decided to do something about it, and while researching how I might manage that /etc/mail/aliases file I discovered that Sendmail supports using a directory server, via the LDAP protocol, to host the alias database. Eureka! I already had OpenLDAP running, and I know how to manage that... why not create a web front-end for editing Sendmail LDAP alias entries?

After a few days of hacking, Sendmail Sobriquet was born.
